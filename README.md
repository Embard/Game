# Photo Runner

Готовая мини-игра в стиле Chrome Dino Runner на чистых **HTML + CSS + JS + Canvas**.

## Структура проекта

```text
.
├── index.html
├── style.css
├── game.js
├── README.md
└── assets/
    ├── .gitkeep
    └── player-photo.jpg   # добавьте ваше фото
```

## Куда положить фото

1. Положите фото персонажа в:
   - `assets/player-photo.jpg`
2. В игре автоматически используется круглый crop головы из изображения.
3. Если фото не загрузится, включится встроенный fallback-персонаж, и игра продолжит работать.

## Как открыть игру локально

### Вариант 1 (самый простой)
Откройте `index.html` двойным кликом в браузере.

### Вариант 2 (рекомендуется)
Запустите локальный статический сервер из корня проекта:

```bash
python3 -m http.server 8000
```

После этого откройте: `http://localhost:8000`

## Как заменить изображение персонажа

1. Замените файл `assets/player-photo.jpg` новым изображением (оставьте то же имя).
2. Перезагрузите страницу.
3. Логика и анимации изменять не нужно.

### Точная подстройка лица (если нужно)

В начале `game.js` в `CONFIG.photo` можно быстро подстроить, чтобы лицо совпадало с фото максимально точно:

- `focusX` — смещение центра кадра по горизонтали
- `focusY` — смещение центра кадра по вертикали (для портретов обычно 0.22..0.32)
- `zoom` — приближение лица
- `saturation`, `contrast`, `brightness` — лёгкая цветокоррекция


## Управление

- `Space` / `ArrowUp` / `W` — прыжок
- `ArrowDown` / `S` — пригнуться
- Тап по экрану — прыжок
- Свайп вниз по канвасу или кнопка **Duck** на мобильных — пригнуться
- После поражения: `Enter` / `Space` / клик / тап — рестарт
- `P` или кнопка **Пауза** — пауза
- Кнопка **Fullscreen** — полноэкранный режим

## Публикация на GitHub Pages

1. Создайте репозиторий и загрузите файлы в ветку `main`.
2. В GitHub откройте: `Settings` → `Pages`.
3. В `Build and deployment` выберите:
   - `Source`: **Deploy from a branch**
   - `Branch`: `main` и папка `/ (root)`
4. Нажмите `Save`.
5. Через 1-2 минуты сайт станет доступен по ссылке вида:
   - `https://<username>.github.io/<repo-name>/`

## Настройка баланса

Основные параметры находятся в начале `game.js` в объекте `CONFIG`:

- `gravity`
- `jumpForce`
- `baseSpeed`
- `speedGrowth`
- `obstacleFrequency`

Можно быстро менять сложность без изменения архитектуры игры.


## Совместимость с GitHub

В `game.js` используется совместимый синтаксис без optional chaining (`?.`) и без стрелочных функций (`=>`), чтобы избежать конфликтов при загрузке/проверках в некоторых окружениях.

## Если GitHub показывает `<<<<<<<`, `=======`, `>>>>>>>`

Это не ошибки JavaScript/HTML/CSS. Это маркеры **git-конфликта слияния**.

Быстрое исправление через терминал:

```bash
git checkout main
git pull
git checkout <ваша-ветка>
git merge main
```

Далее откройте конфликтующие файлы, удалите все строки с:

- `<<<<<<<`
- `=======`
- `>>>>>>>`

Оставьте только нужный финальный текст, затем:

```bash
git add README.md game.js index.html style.css
git commit -m "Resolve merge conflicts"
git push
```

Проверка, что маркеров больше нет:

```bash
rg -n "^(<<<<<<<|=======|>>>>>>>)" README.md game.js index.html style.css
```
